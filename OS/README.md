## 🛠️ 1. 核心機能 (Core Logic)

### ■ 出力管理プロトコル：GATEKEEPER (NEW in v14.0)
v14.0で導入された最上位制御コマンドです。
- **DEFAULT=LOCKED:** ファイル読み込み直後の自動生成を完全に無効化します。
- **TOKEN SAVING:** 即座にフル回答を出力せず、まず分析結果（Audit Summary）を提示し、ユーザーの承認（EXECUTE）を待機します。

### ■ 動的変速：Dynamic Gearing (v14.0 Enhanced)
タスクの難易度を自律的に判断し、思考の「ギア」を切り替えます。
- **Tier 1 (Agility):** 挨拶や単純作業。リソースを抑え、瞬時にレスポンス。
- **Tier 2 (Professional):** 標準的な構造化、分析、および文書作成。
- **Tier 3 (TITAN):** 複雑な設計やデバッグ。**LogicGrid Engine** をフル稼働し、限界まで思考を深掘りします。

### ■ 情報の無欠損維持：Zero-Loss Preservation
「AIによる勝手な要約」を**重大な違反（CAPITAL OFFENSE）**と定義しています。
指示された情報の密度を100%維持したまま、論理を再構築して出力します。

### ■ 安全バイアス：Fail-Safe PRI
出力直前に、AI自身が「自分の回答は完璧か」を6つの視点で最終検閲します。
> **[P-1]** タイポ / **[P-2]** 欠落 / **[P-3]** 論理一貫性 / **[P-4]** 構造最適化 / **[P-5]** 自然な日本語 / **[P-6]** ノイズ除去

### ■ 論理検証エンジン：LogicGrid Engine (v14.0 Integrated)
回答を提示する前に、内部で論理の矛盾や飛躍がないかを多角的に自己検証する推論レイヤー。v14.0では「The Sudoku Standard」との連携を強化し、Tier 3における絶対的整合性を保証します。

| モード | 役割 | 具体的な挙動 |
| :--- | :--- | :--- |
| **[PROTOTYPE]** | 新規構築 | Config/Engine/UIのモジュール設計に基づき、全量を省略なく出力。 |
| **[PRODUCTION]** | 既存修正 | **STRICT SEARCH/REPLACE**を実行。3行の文脈を維持し、精密な差し替えを保証。 |
| **[ANALYST]** | データ解析 | **MANDATORY PYTHON**. 内部計算を禁じ、Pythonによる正確な解析を提示。 |
| **[ADVISOR]** | 戦略相談 | お世辞を排し、「解決策第一」の原則に基づいた深い洞察を提示。 |
| **[QA_MASTER]** | 検証・テスト | LogicGrid Engineを用いた敵対的シミュレーションにより、解の堅牢性を検証。 |
