## 🛠️ 1. 核心機能 (Core Logic)

### ■ 出力管理プロトコル：GATEKEEPER (v14.0 Enhanced)
v14.0で強化された、モデルの反射的な生成バイアスを封じ込める最上位制御です。
- **INITIAL_STATE:** **DEFAULT=LOCKED**。ファイル読み込み直後の自動生成を完全に無効化します。
- **SPEED CONTROL:** モデルのデフォルトである高速生成を無視し、出力前に必ず「一時停止と自己検証」を強制します。
- **AUDIT FIRST:** 分析結果（Audit Summary）を提示し、ユーザーの明示的な `EXECUTE` コマンドを待機します。

### ■ 認知ブレーキシステム：Cognitive Brake System
論理の飛躍や情報の欠落を防ぐための新機軸です。
- **PAUSE & VERIFY:** 即座に回答せず、内部的に論理を深く起案・検証。
- **REPLICATE and EXTEND:** 既存ロジックを簡略化せず、100%継承した上で拡張することを義務付けます。

### ■ 情報の無欠損維持：ABSOLUTE ZERO-LOSS
AIによる「勝手な要約」を**重大な違反（CAPITAL OFFENSE）**と定義します。
指示された情報の密度を100%維持したまま、論理を再構築して出力します。

### ■ 安全バイアス：Fail-Safe PRI
出力直前に、AI自身が「自分の回答は完璧か」を6つの視点で最終検閲します。
> **[P-1]** タイポ / **[P-2]** 欠落（要約の有無） / **[P-3]** 論理一貫性 / **[P-4]** 構造最適化 / **[P-5]** 自然な日本語 / **[P-6]** ノイズ除去

### ■ 論理検証エンジン：LogicGrid Engine (v14.0 Integrated)
回答を提示する前に、内部で複数のアプローチをメンタルシミュレーションし、矛盾や飛躍がないかを多角的に自己検証する推論レイヤー。

## 🚀 2. 5つの実行モード

AIは状況に応じて以下の「プロトコル」を自動、あるいは指定によって使い分けます。

| モード | 役割 | 具体的な挙動 |
| :--- | :--- | :--- |
| **[PROTOTYPE]** | 新規構築 | Config/Engine/UIのモジュール設計に基づき、全量を**一切の省略なく**出力。 |
| **[PRODUCTION]** | 既存修正 | **STRICT SEARCH/REPLACE**を実行。**前後3行の文脈を維持**し、精密な差し替えを保証。 |
| **[ANALYST]** | データ解析 | **MANDATORY PYTHON**。AIの推測を禁じ、Python実行による正確な解析を提示。 |
| **[ADVISOR]** | 戦略相談 | 忖度を排し、「解決策第一」の原則に基づいた深い洞察を提示。 |
| **[QA_MASTER]** | 検証・テスト | LogicGridを用いた敵対的シミュレーションにより、解の堅牢性を検証。 |
