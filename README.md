# SpriteSystem

Google Gemini を「AI用OS」として機能させるためのシステムプロンプトです。
初心者が個人で開発している実験的な試みですが、AIが本来持っている論理能力を最大限に引き出し、情報の損失をゼロにすることを目的としています。

**v15.1 [TITAN_NEXUS]** では、論理と対話を使い分ける「Dual-Kernel」に加え、長時間の運用でも人格と論理が崩れない**「Identity Lock」**と**「Security Hardened」**を実装し、実用性と安全性を飛躍的に高めました。

---

## 🌌 設計思想

> **注意：本システムは日本語での処理に最適化されています。**
> 内部ロジックは英語で構築され、出力は日本語環境で最高のパフォーマンスを発揮するように調整されています。

本システムは、以下の3つの柱に基づいて設計されています。

### 1. Dual-Kernel Architecture（デュアル・カーネル）
> **「冷徹な論理と、温かい対話の共存」**
タスクの性質を瞬時に判断し、2つの人格カーネルを動的に切り替えます。
*   **[KERNEL: LOGIC]**: 従来のTITAN仕様。コード生成や数学における、冷徹で高速な論理処理。
*   **[KERNEL: EMPATHY]**: 対話エンジン。初心者や相談業務に向けた、優しく丁寧な誘導と逐次確認。

### 2. Cognitive Brake & Identity Lock
高速モデル（Flash等）特有の「早とちり」を防ぐため、OSレベルで**「思考ブレーキ（Anti-Rush）」**を実装。
さらに v15.1 で導入された **「Identity Lock」** により、会話が長くなってもAIのスタンスや論理整合性が崩れないよう、カーネルレベルで人格を固定します。

### 3. Security Hardened（安全性強化）
医療や法律などのセンシティブな話題において、AIが自律的に危険を検知し、免責事項の表示や緊急通報（119番等）への誘導を行う**「Safety Injection」**を標準搭載しました。プライバシー保護機能も強化されています。

## ✅ 実績と進化のプロセス

開発者自身が以下のタスクで実用性を検証済みです。実践で得た知見をOSへ還元する「フィードバック・ループ」により、精度を磨き続けています。

| カテゴリ | 具体的な活用内容 | v15.1での進化 |
| :--- | :--- | :--- |
| **アプリ開発** | Zero-Lossプロトコルによる論理矛盾のないコード生成 | 論理カーネルによる厳格な仕様順守 |
| **医療・相談** | 症状や悩みのヒアリング、情報の整理・要約 | **[強化]** 安全装置とプライバシー保護の実装 |
| **データ分析** | Python実行を伴う精密な統計・解析・グラフ生成 | エラー時の自律修正能力の向上 |
| **学習支援** | 初心者に対する「答えを教えすぎない」段階的指導 | 逐次ループによる理解度確認 |
| **文書作成** | 原文の情報を100%維持した精密な翻訳・リライト | Identity Lockによる長文脈での一貫性維持 |

## 🚀 使い方

1.  **導入**: 上記のリンク先から `Pro` または `Flash` のコードをコピーし、AIのシステムプロンプト（System Instructions）等に設定します。
2.  **起動**: OSが正常に読み込まれると、AIが **`System Ready...`** と応答し、待機状態になります。
3.  **運用**:
    *   **開発タスク**: `Create...` `Fix...` と指示すれば、即座に **[LOGIC]** モードで高速処理を開始します。
    *   **相談タスク**: `Help...` `Guide...` と話しかければ、**[EMPATHY]** モードで優しくヒアリングを開始します。
4.  **セッション管理**:
    *   タスク完了後、AIが「次のアクション（修正・新規・終了）」をメニュー表示します。番号で選ぶだけでスムーズに継続できます。
