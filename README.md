# SpriteSystem OS v10.3 | NITAGON Logic Core

### 〜試行錯誤から辿り着いた、思考のOS〜

**SpriteSystem** は、大規模言語モデル（LLM）を「AI用OS」として機能させるためのシステムプロンプト・アーキテクチャです。
数多の検証を繰り返した経験から、「AIが持つ本来の能力を最大限に引き出す」ことを目的に構築されています。

最新の **v10.3** では、新たに自己診断（Self-Diagnostic）と論理再装填（Reload）の概念を導入し、さらなる高精度と客観性を実現しました。

---

## ■ 設計思想：制御と柔軟性の黄金比

本OSは、AIを闇雲に縛るのではなく、**「制御」と「柔軟性・汎用性」の絶妙なバランス**を追求しています。

1. **OS（基盤）とテンプレート（目的）の分離**
   OSは常にクリーンな「思考 of 枠組み」として保ちます。特定の目的を詰め込まないことでAIのポテンシャルを維持し、タスクごとの「テンプレート」を外付けすることで、高度な汎用性を実現します。

2. **客観的な視点による自己分析**
   AIの「肯定バイアス」を抑制するため、あえて一歩引いた客観的な視点で自己評価を行うプロセスを組み込み、アウトプットの論理的整合性を担保します。

3. **ハイブリッド運用による効率化**
   「日常の対話はFlash」「重量級の分析・出力はPro」と、役割の異なる2つのモードを戦略的に使い分け、リソースの最適化と精度の両立を図ります。

---

## ■ v10.3 の核心機能 (Advanced Logic)

* **PHASE 0: SELF-DIAGNOSTIC:** 回答開始前に、AI自身に指示の理解度をセルフチェックさせ、論理の脱線（ドリフト）を未然に防ぎます。
* **DRIFT DETECTION:** 会話が長期化した際の、指示の忘却やブレを自動的に検知・警告します。
* **RELOAD TRIGGER:** システムが不安定になった際、特定のコマンドで論理構造を強制的にリセット・再構築します。
* **ADAPTIVE DIFF STRATEGY:** 修正量に応じて「全置換」と「部分修正」を自動判別し、トークン効率を最適化します。

---

## ■ [RELOAD] システム再装填シーケンス
論理が複雑化した際、以下のプロセスで思考を再定義します。

> **RELOAD TRIGGER:**
> 1. **Current State:** v10.3 Stable
> 2. **Logic Integrity:** Checking... [OK]
> 3. **Self-Diagnostic:** AI Advisor / Data Analyst / Prompt Engineer modes ready.
> 思考の枠組みを最新の状態に固定しました。

---

## ■ 推奨される開発サイクル

1. **【特化】Flashモードでの試行錯誤** AIを目的の方向に特化させ、正解の糸口を掴みます。
2. **【実行】Proモードによる高度な出力** 分析や正確なコード出力が必要なフェーズで移行し、重量級の作業を完遂させます。
3. **【調整】差分更新による仕上げ** 納得がいくまで、必要な箇所だけを入れ替える微調整を繰り返します。
4. **【資産化】テンプレートとして残す** 成功プロセスを「テンプレート」として保存。次回からは一瞬で最高精度を再現可能になります。
