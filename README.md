# SpriteSystem
### 〜試行錯誤から辿り着いた、思考のOS (v10.0)〜

SpriteSystem は、Google Gemini を「AI用OS」として機能させるためのシステムプロンプト・アーキテクチャです。
数多の生成AIでプロンプト検証を繰り返した経験から、「AIが持つ本来の能力を最大限に引き出す」ことを目的に構築されています。

---

## ■ 設計思想：制御と柔軟性の黄金比

本OSは、AIを闇雲に縛るのではなく、**「制御」と「柔軟性・汎用性」の絶妙なバランス**を追求しています。

1. **OS（基盤）とテンプレート（目的）の分離**
   OSは常にクリーンな「思考の枠組み」として保ちます。特定の目的を詰め込まないことで、AIのポテンシャルを維持。タスクごとの「テンプレート」を外付けすることで、高度な汎用性を実現します。

2. **客観的な視点による自己分析**
   AIの「肯定バイアス」を抑制するため、あえて一歩引いた客観的な視点で自己評価を行うプロセスを組み込み、アウトプットの論理的整合性を担保します。

3. **ハイブリッド運用による効率化**
   「日常の対話はFlash」「重量級の分析・出力はPro」と、役割の異なる2つのモードを戦略的に使い分け、リソースの最適化と精度の両立を図ります。

---

## ■ 推奨される開発サイクル（初めてのタスクに挑む場合）

1. **【特化】Flashモードでの試行錯誤**
   まずは Flash との対話を通じ、AIを目的の方向に特化させます。この段階では柔軟性を重視し、AIと共に正解の糸口を掴みます。

2. **【実行】Proモードによる高度な出力**
   分析や正確なコード出力が必要なフェーズでは Pro へ移行。Flashで固めた文脈を引き継ぎ、最もパワーが必要な「しっかりとした作業」を完遂させます。

3. **【調整】差分更新による仕上げ**
   納得がいくまで、必要な箇所だけを入れ替える「差分更新」で微調整を繰り返します。

4. **【資産化】最後にテンプレートとして残す**
   納得できる成果が得られたら、その成功プロセスを「テンプレート」として保存。これにより、次回からは迷わず一瞬で最高精度を再現できる、自分専用の仕様へと進化します。

## ■ 実績と検証

開発者自身が、複雑な「数独パズル生成エンジン」の開発を通してこのフローの有効性を検証済みです。
「対話による特化 → Proでの一括出力 → 成功パターンのテンプレート化」という流れを用いることで、停滞を乗り越え、多機能なシステムを1ファイルにまとめ上げることに成功しました。
