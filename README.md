# SpriteSystem

Google Gemini を「AI用OS」として機能させるためのシステムプロンプトです。
初心者が個人で開発している実験的な試みですが、AIが本来持っている論理能力を最大限に引き出し、情報の損失をゼロにすることを目的としています。

---

## 🌌 設計思想

> **注意：本システムは日本語での処理に最適化されています。**
> 内部ロジックは英語で構築され、出力は日本語環境で最高のパフォーマンスを発揮するように調整されています。

本システムは、以下の3つの柱に基づいて設計されています。

### 1. OS（基盤）とテンプレート（目的）の分離
> **「OSそのものに特定の目的を詰め込んで更新しない」**
基盤（OS）は常にクリーンに保ち、タスクごとに「外付けテンプレート」を運用することで、プロンプトの肥大化による精度低下を防ぎ、高いパフォーマンスを維持します。

### 2. 「GATEKEEPER」と「動的変速」による制御
v14.0で導入された**GATEKEEPER**プロトコルにより、AIのデフォルトの高速生成を抑制。同時に、タスクの難易度に応じて思考の深さを切り替える**「動的変速（Dynamic Gearing）」**を搭載し、情報の無欠損（Zero-Loss）と快適なレスポンスを両立させました。

### 3. LogicGrid Engine による論理整合性
指示を曖昧にせず、LogicGridによるステップバイステップの検証を強制します。重要な仕事（デバッグや数学的証明など）では、AIが自動的にPythonを実行して客観的な正確性を担保する**「The Sudoku Standard」**を組み込んでいます。

## ✅ 実績と進化のプロセス

開発者自身が以下のタスクで実用性を検証済みです。実践で得た知見をOSへ還元する「フィードバック・ループ」により、精度を磨き続けています。

| カテゴリ | 具体的な活用内容 |
| :--- | :--- |
| **高度プロンプト設計** | LogicGridを用いた高密度・高精度な指示書の構築 |
| **データ分析** | Python実行を伴う精密な統計・解析・グラフ生成 |
| **ディスカッション** | 忖度を排した「冷徹な外部知性」としての多角的視点の提供 |
| **文書作成** | 原文の情報を100%維持した精密な翻訳・リライト |
| **アプリ作成** | Zero-Lossプロトコルによる論理矛盾のないコード生成 |

## 🚀 使い方

1.  **導入**: 上記のリンク先から `Pro` または `Flash` のコードをコピーし、AIのシステムプロンプト（System Instructions）等に設定します。
2.  **起動**: OSが正常に読み込まれると、AIが **`System Ready. 指示を待機中。`** と応答し、出力ゲートが凍結（待機状態）されます。
3.  **運用**: 
    * 依頼を投げると、まずAIが **Audit Summary（分析レポート）** を提示します。
    * 内容を確認し、問題がなければ **`EXECUTE`** または **`OK`** と指示してください。本実行が開始されます。
