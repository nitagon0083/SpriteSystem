# SpriteSystem

Google Gemini を「AI用OS」として機能させるためのシステムプロンプトです。
初心者が個人で開発している実験的な試みですが、AIが本来持っている論理能力を最大限に引き出し、情報の損失をゼロにすることを目的としています。

---

## 🌌 設計思想

> **注意：本システムは日本語での処理に最適化されています。**
> 内部ロジックは英語で構築され、出力は日本語環境で最高のパフォーマンスを発揮するように調整されています。

本システムは、以下の3つの考え方に基づいて設計されています。

### 1. OS（基盤）とテンプレート（目的）の分離
> **「OSそのものに特定の目的を詰め込んで更新しない」**

OSに指示を詰め込みすぎると精度が低下します。基盤（OS）は常にクリーンに保ち、タスクごとに「外付けテンプレート」を運用することで、高いパフォーマンスを維持します。

### 2. 「GATEKEEPER」による厳格な出力管理
v14.0で導入された新機能です。AIのデフォルトの高速生成（High-speed generation）を抑制し、出力前に必ず検証プロセスを挟みます。これにより、モデル特有の「要約癖」を排除し、情報の無損失（Zero-Loss Preservation）を実現しました。

### 3. LogicGrid Engine による論理整合性
指示を曖昧にせず、LogicGridによるステップバイステップの検証を強制します。重要な仕事（プログラムの修正や数学的証明）の時は、AIが自動的にPythonを実行して検証を行うなど、客観的な正確性を担保する**「The Sudoku Standard」**を組み込んでいます。

## 🚀 使い方

1.  **導入**: 上記のリンク先から `Pro` または `Flash` のコードをコピーし、AIのシステムプロンプト等に設定します。
2.  **起動**: OSが読み込まれたら、AIが「待機中」であることを示します。
3.  **運用**: 最初に出力される **Audit Summary（分析レポート）** を確認し、問題がなければ `EXECUTE` と指示して実行させます。
