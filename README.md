# SpriteSystem

Google Gemini を「AI用OS」として機能させるためのシステムプロンプトです。
初心者が個人で開発している実験的な試みですが、AIが本来持っている論理能力を最大限に引き出し、情報の損失をゼロにすることを目的としています。

v15.0 では、従来の強力な論理エンジンに加え、対話の柔軟性と安全性を統合した **「Dual-Kernel（デュアル・カーネル）」アーキテクチャ** へと進化しました。

---

## 🌌 設計思想

> **注意：本システムは日本語での処理に最適化されています。**
> 内部ロジックは英語で構築され、出力は日本語環境で最高のパフォーマンスを発揮するように調整されています。

本システムは、以下の3つの柱に基づいて設計されています。

### 1. Dual-Kernel Architecture（デュアル・カーネル）
> **「冷徹な論理と、温かい対話の共存」**
タスクの性質を瞬時に判断し、2つの人格カーネルを動的に切り替えます。
*   **[KERNEL: LOGIC]**: 従来のTITAN仕様。コード生成や数学における、冷徹で高速な論理処理。
*   **[KERNEL: EMPATHY]**: 新搭載の対話エンジン。初心者や相談業務に向けた、優しく丁寧な誘導と逐次確認。

### 2. 「Cognitive Brake」による暴走制御
高速モデル（Flash等）特有の「早とちり（ハルシネーション）」を防ぐため、OSレベルで**「思考ブレーキ（Anti-Rush Protocol）」**を実装しました。
曖昧な指示に対しては一気に回答せず、**Sequential Loop（逐次質問）**を用いて一歩ずつ確実にゴールへ導きます。

### 3. LogicGrid Engine による論理整合性
指示を曖昧にせず、ステップバイステップの検証を強制します。重要な仕事（デバッグや数学的証明など）では、AIが自動的にPythonを実行して客観的な正確性を担保する**「The Sudoku Standard」**を組み込んでいます。

## 🚀 使い方

1.  **導入**: 上記のリンク先から `Pro` または `Flash` のコードをコピーし、AIのシステムプロンプト（System Instructions）等に設定します。
2.  **起動**: OSが正常に読み込まれると、AIが **`System Ready...`** と応答し、待機状態になります。
3.  **運用**: 
    * **開発タスクの場合**: `Create...` `Fix...` と指示すれば、即座に **[LOGIC]** モードで高速処理を開始します。
    * **相談タスクの場合**: `Help...` `Guide...` と話しかければ、**[EMPATHY]** モードで優しくヒアリングを開始します。
4.  **セッション管理**:
    * タスク完了後、AIが「次のアクション（修正・新規・終了）」をメニュー表示します。番号で選ぶだけでスムーズに継続できます。
