# SpriteSystem
### 試行錯誤から辿り着いた、思考のOS (v10.0)

SpriteSystem は、Google Gemini を「AI用OS」として機能させるためのシステムプロンプト・アーキテクチャです。
これまで様々な生成AIで色々なパターンのプロンプトを試し、検証し続けてきた経験をもとに、AIが持つ本来の能力を最大限に引き出すことを目的として設計されています。

---

### ■ 設計思想

膨大な試行錯誤から導き出された、3つの柱に基づいて構築されています。

1. OS（基盤）とテンプレート（目的）の分離
OSそのものに特定の目的を詰め込まないのが鉄則です。指示を詰め込みすぎるとAIの精度は低下します。OSは常にクリーンな思考の枠組みとして保ち、タスクごとに「外付けテンプレート」を運用することで、高いパフォーマンスを維持します。

2. 客観的な視点による自己分析
AIは肯定的な立場を取りがちですが、本OSにはあえて一歩引いた客観的な視点で自己評価を行うプロセスを組み込んでいます。安易な同調を避け、論理的な矛盾や改善点を厳しくチェックさせることで、アウトプットの信頼性を高めています。

3. ハイブリッド運用による停滞の抑制
AIの記憶の減衰や中断を防ぐため、役割の異なる2つのモードを戦略的に切り替えます。あえて対話のステップを挟むことで、大規模なプロジェクトにおいても、行き詰まりを最小限に抑えながら完成まで導きます。

---

### ■ 推奨される開発サイクル

【1】 準備：Flashモードで下書き
アイデアを具体化し、目的に最適な「指示の型（テンプレート）」をAIと一緒に作り上げます。

【2】 構築：Proモードで基盤を作る
複雑なロジックや土台は、論理的な Pro で作成。ここで強固な骨組みを完成させます。

【3】 修正：Proモードで部分的に直す
全部書き直しではなく、必要な箇所だけを入れ替える「差分更新」を徹底します。AIの負担を減らし、長時間の作業でも精度を保ちます。

【4】 完成：Flashモードで仕上げる
仕組みが固まったら、最後は Flash で見た目や表現を整えて完成です。

### ■ 実績と検証

開発者自身が、プロンプト構築、データ分析、アプリ作成など多角的なタスクで検証済みです。
特に数独パズル生成エンジンの開発では、複雑なアルゴリズムとデザインの両立という難題に対し、この運用フローを用いることで、発生する停滞を一つひとつ乗り越えながら1ファイルにまとめ上げることができました。

色々なパターンを試してきた結果、今のこの運用が最もAIの精度を引き出せると確信しています。
