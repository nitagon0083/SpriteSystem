# SpriteSystem

A system prompt architecture designed to transform Google Gemini into a "Logical OS for AI."

This is an experimental project developed by a solo creator. It aims to maximize the inherent capabilities of AI to their fullest potential.

> **Note: This system is optimized for processing in Japanese.**
> While multilingual input is supported, the internal logic and response structures are tuned for peak performance within a Japanese language environment.

This system is built upon the following three core principles:

### 1. Decoupling of OS (Core) and Template (Objective)
> **"Do not bloat the OS itself with specific task-oriented instructions."**

This is the most critical rule of the system. Overloading the OS with specific instructions leads to a decline in precision. By keeping the core (OS) clean and utilizing "external templates" for each task, the system maintains high-level performance.

### 2. Comfort through "Dynamic Gearing"
A new feature introduced in v13.4. The AI autonomously determines the complexity of the taskâ€”whether it is a simple greeting or a complex technical problem. By switching "gears" to provide quick answers for simple tasks and deep, concentrated logic for difficult ones, it ensures a stress-free user experience.

### 3. Flexibility and Precision Balance
The system values "flexibility," allowing the AI to think freely without being overly constrained by rigid instructions. However, for critical tasks (e.g., code debugging), the AI automatically switches to a high-precision mode via the **"Fail-Safe Bias,"** ensuring the most appropriate response for the situation.
